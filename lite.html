BannerHub Lite

The lightweight member of the family. BannerHub Lite layers the GOG / Epic / Amazon store tabs, a Winlator HUD, Wine Task Manager, Component Manager and per-game performance tooling onto the slim GameHub Lite base — ideal for lower-end or storage-constrained devices.

What's in BannerHub Lite

🎮 Game store tabs

  • GOG — OAuth login, library cache + search, 3 view modes (List / Grid / Poster), SteamGridDB cover art, A-Z sort, Gen 2/1/installer download pipeline, exe picker, Set .exe, Copy to Downloads, uninstall.
  • Amazon — PKCE OAuth, GetEntitlements library sync, manifest.proto (LZMA/XZ), 6-parallel downloads with SHA-256, FuelPump launch + SDK DLLs.
  • Epic — OAuth login, catalog sync, chunked manifest downloads (Fastly/Akamai), 6-parallel with speed display, exe picker & launch.

🧰 In-game sidebar

  • Wine Task Manager — Applications / Processes / Launch tabs, Kill buttons, Container Info (CPU, RAM, VRAM, device, Android), 3s auto-refresh, and a WINEPREFIX file browser that launches .exe/.msi/.bat/.cmd directly.
  • Winlator HUD overlay — API draw / GPU% / CPU% / RAM% / battery / temp / FPS + rolling graph, with a 0–100% opacity slider.
  • Performance toggles (root) — Sustained Performance Mode + CPU performance governor, and Max Adreno Clocks (KGSL min-freq = max).
  • AI Frame Generation (v1.0.2+) — frame-gen menu with 6 presets, multiplier & flow-scale, ported from BannerHub 3.7.x.

⚙️ Components

  • Component Manager — color-coded cards; Add / Inject / Backup / Remove / Remove All; supports .wcp (zstd & XZ tar) and .zip GPU drivers.
  • Online downloader — multiple repositories incl. The412Banner Nightlies, with three-level Repo → Type → Asset navigation.
  • BCI launcher button — toolbar shortcut to BannersComponentInjector.

🔧 Settings & system

  • VRAM Limit Unlock — adds 6 / 8 / 12 / 16 GB to the per-game VRAM picker.
  • Per-game CPU core affinity — Efficiency / Performance / Prime core selection.
  • GPU System Driver default, Offline Steam skip, Launch Fix (hardware-whitelist bypass), explicit Grant Root Access.
  • Export / Import config — per-game settings as JSON, fully cross-compatible with BannerHub. Japanese localization (3,534 strings).
🧬 Inherited from the base

Built on GameHub Lite 5.1.4

BannerHub Lite starts from GameHub Lite by Producdevity — itself a community patch of XiaoJi's GameHub 5.1.0. A lot of what makes Lite "lite" comes from that base; BannerHub adds the store tabs and tooling on top. Here's the honest split:

Comes from GameHub Lite

  • Telemetry & tracking removed at the base — Umeng analytics, Firebase, JPush, Tencent login, crash reporting.
  • No account requirement — login removed.
  • Bloat removed — Cloud Streaming, Xbox Cloud Gaming and social features stripped (≈118 MB → 52 MB at the base).
  • Trimmed permissions & offline support.
  • RTS Touch Controls — tap-to-click, box-select, long-press right-click, two-finger pan, pinch-zoom (added in GameHub Lite 5.1.4 by @Nightwalker743).
  • gamehub.lite package rename + a custom community news page.

…and from GameHub itself

  • The Wine + Box64 / FEX runtime that actually runs Windows games.
  • Container / Wine-prefix management and the in-game sidebar.
  • Native Steam client integration.
  • The base DXVK / VKD3D / Box64 component pipeline.

BannerHub Lite then adds: the GOG / Epic / Amazon store tabs, Component Manager + online downloader, Wine Task Manager, Winlator HUD, AI Frame Generation, VRAM unlock, CPU affinity, config export/import and more (see above).

Release history

Every notable feature and fix, newest first. Each version links to its full GitHub release notes.

Known issue (current): Proton 11 can hang on "Initializing…" when launching Steam games with a Steam client. Workaround: use a different Wine/Proton component for Steam-client launches until resolved.

v1.0.22026-05-12release notes ↗
  • AI Frame Generation menu — new "Frame Generation" row in the in-game Controls sidebar with 6 presets (ECO/FLOW/BAL/BOOST/CLEAR/MAX), 2×/3×/4× multiplier, flow-scale slider and an FPS limiter; settings persist in bh_framegen.
  • Per-package ICD JSON so frame-gen works on every variant package without per-variant patching.
  • Server-side firmware fix — the Add-Game path now serves imagefs 1.3.7 (fixes the "Failed to initialize the simulator" error from v1.0.1, including for older installs — no client change needed).
v1.0.12026-05-12release notes ↗
  • First stable on the BannerHub API — all 9 variants moved to bannerhub-api.the412banner.workers.dev; components, firmware and catalog served from BannerHub.
  • Firmware bumped to 1.3.7, aligning 5.x (Lite) and 6.0 (BannerHub 3.7.x) clients on the same firmware.
  • HID-mode launch dialog fix — the broken touch/HID picker backported into 5.1.4 is rewritten to a no-op so games launch directly.
